<h2 style="margin-top: 30px;">III. Methodology</h2>
 본 연구에서는 영상 내 객체를 실시간으로 탐지하고 추적하기 위해 딥러닝 기반의 1단계 객체 탐지모델인 YOLO(You Only Look Once) 알고리즘을 채택하였다. 특히, 드론 뷰와 같이 객체의 회전각이 다양하게 나타나는 환경에서 탐지 정확도를 극대화하기 위해 OBB(Oriented Bounding Box) 기술이 적용된 YOLO 모델을 활용하여 시스템을 구축하였다.
 
- 객체 탐지 알고리즘 (YOLO Architecture)

 YOLO는 입력 이미지를 S×S 크기의 그리드로 분할하고, 단일 합성곱 신경망(CNN)을 통해 각 그리드 셀에서 바운딩 박스의 좌표 및 클래스 확률을 동시에 예측하는 단일 경로 아키텍처를 가진다. 기존의 R-CNN 계열 모델들이 영역 제안과 분류를 분리하여 처리하는 것과 달리, YOLO는 객체 탐지를 하나의 회귀 문제로 재정의하여 연산 속도를 비약적으로 향상시켰다. 본 프로젝트에서는 YOLO 아키텍처의 빠른 추론 속도를 활용하여 동영상 프레임 단위의 실시간 객체 추적을 구현하였다.
- OBB (Oriented Bounding Box)의 구조적 특징
  
 일반적인 객체 탐지 모델은 이미지의 x, y축에 평행한 직사각형 형태의 HBB(Horizontal Bounding Box)를 생성한다. 그러나 HBB는 드론이나 위성 영상과 같이 수직 시점에서 촬영된 객체나 밀집된 객체를 탐지할 때, 불필요한 배경 노이즈를 포함하거나 객체 간의 바운딩 박스가 중첩되는 치명적인 한계가 있다.
 
 이러한 문제를 해결하기 위해 본 연구에서는 회전형 바운딩 박스(OBB) 기법을 도입하였다. OBB 모델은 기존 HBB의 중심점 좌표와 너비, 높이 정보에 회전 각도 파라미터를 추가하여 (x_c,y_c,ω,h,θ) 형태로 객체를 예측한다. Ultralytics YOLO OBB 모델의 내부 연산에서 이 각도 파라미터는 라디안 단위로 저장되며, [-π/4,3π/4) 범위로 정규화되어 처리된다. 이 수학적 접근을 통해 객체의 실제 형태와 방향성에 정확히 일치하는 바운딩 박스를 생성함으로써, 객체 간의 간섭을 최소화하고 IoU(Intersection over Union) 측정의 정밀도를 향상시켰다.
- 전이 학습 및 하이퍼파라미터 최적화

 실제 환경에서의 추론 성능을 확보하기 위해, 본 연구에서는 밑바닥부터 모델을 학습시키는 대신 대규모 데이터셋(DOTAv1 등)으로 사전 학습된 가중치 모델(예: yolov8n-obb.pt)을 활용하는 전이 학습 기법을 적용하였다. 또한, 영상의 특성 및 타겟 객체의 크기에 따라 최적의 성능을 도출하기 위해 다음과 같은 하이퍼파라미터 튜닝을 수행하였다. 
클래스 필터링: 모델이 인식할 수 있는 다수의 클래스 중, 목적에 맞는 특정 객체(예: 차량을 의미하는 클래스 ID 9, 10)만을 타겟팅하여 불필요한 연산을 줄이고 목적 객체의 추적 연속성을 높였다.

 신뢰도 임계값 조절: 모델의 예측 확신도 커트라인(conf 파라미터)을 영상 환경에 맞춰 동적으로 조정(예: 0.15)함으로써, 형태가 일부 가려지거나 크기가 작은 소형 객체의 미탐(False Negative) 비율을 감소시켰다.
 
 이러한 딥러닝 기반 탐지 및 최적화 프로세스를 통해 영상 내에서 움직이는 객체를 안정적이고 독립적으로 식별(Track ID 부여)하고 추적할 수 있는 시스템을 완성하였다.

<h2 style="margin-top: 30px;">IV. Evaluation & Analysis</h2>
 
- 훈련 손실의 스파이크 현상과 학습 불안정성
  
 train/box_loss 및 train/cls_loss 곡선 전반에서 하강 추세 중간마다 급격하게 튀어 오르는 스파이크 현상이 다수 관찰되었다. 이는 모델이 가중치를 업데이트하는 과정에서 지역 최솟값을 탈출하기 위해 학습률이 변동되었거나, 모자이크 등 강력한 데이터 증강 기법이 적용된 어려운 형태의 미니 배치를 마주했을 때 나타나는 전형적인 현상이다. 전반적인 우하향 추세는 유지하고 있으나, 모델이 훈련 데이터의 복잡성에 완벽히 적응하기에는 다소 불안정한 학습 과정을 겪었음을 시사한다.
- 검증 손실의 지수적 증가와 과적합 발생

 가장 주목해야 할 부분은 val 그래프의 궤적이다. 훈련 손실이 감소함에도 불구하고, val/box_loss와 val/cls_loss 곡선이 학습 중반부 이후 일정 지점에서 최저점을 찍은 뒤 지수적으로 다시 반등하여 우상향하는 U자형 개형을 뚜렷하게 보이고 있다. 이는 모델이 훈련 데이터의 노이즈와 특수한 패턴까지 과도하게 암기하기 시작하여, 처음 보는 검증 데이터에 대한 예측 성능이 오히려 훼손되고 있음을 보여주는 명백한 과적합의 증거이다.
- 진단 결과 및 최적 가중치(best.pt)의 중요성

이러한 지수적 반등은 현재 설정된 모델(예: Medium 모델 등)의 수용력이 주어진 데이터셋의 규모에 비해 지나치게 크거나, 학습 에폭이 과도하게 길게 설정되었을 때 흔히 발생한다. 따라서 최종 시스템 테스트 및 평가를 진행할 때는 마지막 에폭의 가중치(last.pt)가 아닌, 과적합이 시작되기 직전 즉, Validation Loss가 가장 낮았던 시점(곡선의 최저점)에서 자동으로 저장된 best.pt 가중치를 반드시 사용해야만 실환경(CCTV 동영상 등)에서 신뢰도 높은 교통량 계수 성능을 보장할 수 있다.
- F1-Score 최적화
  
BoxF1_curve.png는 모델의 정밀도와 재현율의 조화 평균인 F_1 점수를 보여준다. F_1=2*(precision*recall)/(precision+recall) 그래프의 정점이 위치한 지점의 Confidence 값이 바로 이 시스템의 수학적 최적 효율점이다. 코드 구동 시 main.py의 conf 파라미터를 이 정점의 수치로 설정함으로써, 시스템의 인지 부하(노이즈 탐지)를 최소화하면서도 탐지 누락을 방어할 수 있다.
- 종합 성능 지표
  
 BoxPR_curve.png에서 곡선 아래 면적(AUC)으로 도출된 mAP_50 (Mean Average Precision) 지표는 모델이 IoU 50% 기준에서 객체를 얼마나 완벽하게 추적하고 있는지를 대변하며, 곡선이 우상단에 밀착할수록 우수한 탐지력을 가짐을 확인하였다.
 
 confusion_matrix_normalized.png는 모델이 예측한 차종과 실제 정답 간의 일치도를 백분율로 시각화한 지표이다.
 
 대각선 성분의 집중도: 행렬의 대각선 값이 1.0에 가까울수록 해당 클래스(예: 대형 차량, 소형 차량)를 정확히 진단해 냈음을 의미한다.
- 오분류 경향성 진단

만약 대형 차량을 소형 차량으로 잘못 판단한 비율이 높거나, 배경을 객체로 착각한 비율이 존재한다면 이는 모델의 시각적 한계점을 시사한다. 본 결과에서는 배경 오탐지 비율이 낮게 나타나, 불필요한 연산 낭비 없이 핵심 타겟만을 선별하는 우수한 필터링 능력이 입증되었다.

- 크기 편향성 및 공간 스케일링

 훈련 데이터 내 차량 바운딩 박스의 너비와 높이 분포가 특정 크기에 밀집되어 있는지 분석하였다. 밀집도가 높을 경우, 카메라의 고도나 화각 변화에 따른 스케일 변화에 취약할 수 있으므로 현장 적용 시 입력 영상의 리사이징 최적화가 필수적이다.
- 회전각 학습

 차량들이 도로의 주행 방향을 따라 일정한 벡터 각도를 띠고 있는 것을 확인하였다. 모델은 이 각도 데이터를 성공적으로 역전파하여, 교차로 등에서 차체 방향이 비스듬하게 틀어지더라도 박스가 차량의 실제 픽셀 형태에 완벽하게 밀착하는 기하학적 추론 능력을 확보하였다.
- 데이터셋의 클래스 불균형
  
 train_batch1, train_batch2, train_batch40, train_batch41, train_batch42 이미지를 분석한 결과, 해당 데이터셋은 해상 부유물(선박류), 항만 시설, 테니스 코트, 야구장 등 특정 인프라 중심으로 구성되어 있음을 확인하였다.
- 데이터 증강 기법 적용
  
훈련 배치 이미지들은 원본 이미지를 그대로 사용하지 않고, 4개의 서로 다른 이미지를 하나의 프레임에 짜깁기하는 모자이크 데이터 증강 기법이 성공적으로 적용되어 있음을 보여준다. 이 기법은 모델이 다양한 스케일과 잘린 객체에 적응하도록 유도하여 앞서 확인한 results.png에서의 일반화 성능 확보에 기여한 것으로 평가된다.
- 우수한 선박 방향성 예측
  
정답지와 비교했을 때, 모델은 해상에 밀집된 소형 선박 객체들을 빠짐없이 탐지하였을 뿐만 아니라, 선박의 선수가 향하는 대각선 방향에 맞춰 바운딩 박스(OBB)를 정확하게 회전시켜 포개어 놓았다.
- 밀집 객체 분리력
  
항구에 나란히 정박해 있는 수십 대의 선박들을 하나의 덩어리로 인식하지 않고, 개별 객체로 분리해 내는 뛰어난 해상력을 보여준다. 이는 비최대 억제(NMS, Non-Maximum Suppression) 파라미터가 적절히 작동하고 있음을 반증한다.
- 속도 벡터와 회전각 동기화
  
주행 중인 차량들의 이동 궤적을 프레임 단위로 추적할 때, OBB가 차량의 진행 방향과 완벽하게 평행을 유지하고 있다. 차선 변경 시에도 박스의 형태가 불필요하게 커지지 않고 객체의 실제 픽셀 면적에 타이트하게 밀착되어, 공간 인식의 효율을 극대화하였다.
- 소형 차량 추적 안정성
  
영상 전반에 걸쳐 등장하는 승용차들(예: 화면 상단의 빨간색 승용차 id:1, 하단의 파란색 승용차 id:4 등)은 프레임이 빠르게 전환되어도 ID(고유 번호)가 변하지 않고 연속성을 훌륭하게 유지하고 있다. 이는 모델이 소형 객체의 시공간적 특성을 안정적으로 연산하고 있음을 증명한다.
- 대형 화물차의 파편화 및 미탐 현상
  
중앙 차선을 주행하는 긴 흰색 트레일러 트럭의 추적 결과에서 모델의 치명적인 한계가 관찰된다. 트럭이 화면 중앙을 통과할 때 모델이 이를 아예 인식하지 못하는 인지 오류가 발생했다.
- 오류 발생의 데이터적 역학
  
이는 나노 모델(yolov8n-obb)의 연산 수용력 한계이기도 하지만, 근본적으로는 특이 케이스에 대한 학습 데이터의 결핍 때문이다. 모델이 일반적인 승용차나 버스의 규격은 학습했으나, 비정상적으로 길거나 관절이 꺾이는 대형 트레일러의 기하학적 특징은 온전히 학습하지 못하여 발생하는 전형적인 분류 실패 사례이다.
